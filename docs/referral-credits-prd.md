# PRD: Referral Programme & Credits System
**QuickList AI** · Last updated: March 2026

---

## Problem Statement

QuickList's free tier (5 generations/month) is the natural entry point, but there is no mechanism to reward users who bring others in, no way for users to earn more generations without paying, and no viral loop baked into the product. Word-of-mouth is the highest-leverage growth channel for a tool aimed at casual sellers — these people live in Facebook groups, Vinted communities, and car boot WhatsApp chats. The product currently has no hook into those networks.

---

## Goals

**User goals**
- Earn more AI generations without immediately upgrading to paid
- Share a simple code or link with friends who sell online
- See clearly how many credits they have and how they earned them

**Business goals**
- Achieve a referral rate of ≥15% of new signups via referral code within 90 days of launch
- Reduce cost-per-acquisition below £2 via organic referral vs. paid channels
- Increase free→paid conversion by 10% among users who have referred at least one person (referred users are more engaged and more likely to convert)
- Create a viral coefficient above 0.3 (every 10 users bring in 3+ more)

---

## Non-Goals

- **Paid affiliate programme** — cash payouts to referrers are out of scope for v1. Credits only.
- **Double-sided paid incentives** — we're not paying referred users cash, only bonus generations.
- **Referral analytics dashboard** — a full dashboard is P2. v1 shows basic count + credit balance.
- **Team/organisation referrals** — B2B mechanics out of scope. This is consumer/prosumer only.
- **Referral expiry** — codes don't expire in v1. Credits expire after 12 months (handled by DB field).

---

## How It Works

### The full journey, plainly stated

Every new user gets a unique referral code the moment their account is created — something like `DEAN47` (first name + random 2 digits). This happens automatically, they don't have to do anything.

Users can find their code and shareable link in a new "Refer & Earn" section in their account settings. The link looks like `quicklist.ai/?ref=DEAN47`. They can copy it, share it anywhere — text, WhatsApp, Facebook group, Vinted bio.

When someone clicks that link, the `ref` code is saved in `localStorage`. If they sign up within 30 days, the code is submitted automatically as part of the Clerk webhook flow. The new user immediately gets **+5 bonus generations** credited to their wallet. The referrer gets **+3 credits** credited at the same moment.

Credits sit in a wallet. When a user hits Generate, the system checks their credit balance first — if they have credits, one is deducted. If credits are zero, it falls through to the monthly quota check. Monthly quota still resets on the 1st of each month. Credits do not reset.

**Milestone reward**: When a user's successful referral count hits 3, they automatically receive a free month of the Starter tier (50 generations). This is a one-time reward, tracked with a flag on the user record.

**Viral loop (optional, P1)**: The first listing generated by a user who signed up via referral (and is still on the free tier) includes a soft watermark in the downloaded ZIP/text: *"Made with QuickList AI — try it free: quicklist.ai/?ref=THEIROWN CODE"*. This is removed once they're on a paid plan or have referred 1 person themselves.

## User Stories

1. As a new user who followed a referral link, I want to automatically receive bonus generations on signup so that I can try the product more fully before deciding whether to pay.

2. As an existing user, I want to find my unique referral link in one tap so that I can share it in a Vinted group or WhatsApp without having to ask support for it.

3. As an existing user, I want to see how many credits I have and how many successful referrals I've made so that I know how close I am to the milestone reward.

4. As an existing user who has referred 3 people, I want to automatically receive a free month of Starter tier so that I'm rewarded without having to claim anything manually.

5. As a user running low on monthly quota, I want my credits to top up my available generations automatically so that I don't hit a wall mid-session.

6. As a user on a paid plan, I still want to accumulate referral credits so that they stack against future billing periods (or unlock early renewal).

## Requirements

### P0 — Must ship

**Referral code generation**
- Every user gets a code automatically on account creation via Clerk webhook
- Code format: uppercase first name (max 6 chars) + 2-digit random number, e.g. `SARAH42`
- Code must be unique across all users — retry on collision
- Stored in `users` table as `referral_code VARCHAR(12) UNIQUE`

*Acceptance criteria*
- [ ] New user record always has a non-null `referral_code` after Clerk `user.created` webhook fires
- [ ] No two users share the same code
- [ ] Code is URL-safe (uppercase alpha + digits only)

---

**Referral link capture**
- Landing page JS stores `?ref=CODE` value in `localStorage` key `ql_ref` on page load
- Value persists for 30 days (stored with timestamp, cleared if >30 days old)
- On Clerk signup completion, frontend reads `localStorage.ql_ref` and POSTs it to `/api/referral/complete`

*Acceptance criteria*
- [ ] Visiting `/?ref=DEAN47` sets `localStorage.ql_ref = "DEAN47"` with a timestamp
- [ ] Revisiting with no `?ref` param does NOT clear an existing stored code
- [ ] A code stored >30 days ago is treated as expired and not submitted
- [ ] Code is submitted once only — cleared from localStorage after successful POST

---

**Credit wallet**
- New table `user_credits` (see Schema below)
- `GET /api/credits` returns balance and transaction history
- Credits are used before monthly quota in `/api/generate`
- Credits do not reset monthly

*Acceptance criteria*
- [ ] User with 3 credits + 5 monthly quota remaining: generate uses 1 credit first, balance becomes 2
- [ ] User with 0 credits + 5 monthly quota: uses monthly quota normally
- [ ] User with 0 credits + 0 monthly quota: gets 429 "quota exceeded" as today
- [ ] Credit deduction and generation both succeed or both roll back (transaction)

---

**Referral completion + reward issuance**
- `POST /api/referral/complete` validates the code, creates referral record, issues credits to both parties
- Referred user: +5 credits
- Referrer: +3 credits
- Duplicate prevention: a user can only be "referred" once

*Acceptance criteria*
- [ ] Valid code submitted by a brand new user: both parties receive correct credits
- [ ] Invalid/nonexistent code: 400 error, no credits issued
- [ ] User submitting their OWN code: 400 error, rejected
- [ ] User who already had a referral applied: 409 error, no double reward
- [ ] Referral record created with `status = 'complete'` and `rewarded_at` timestamp

---

**Refer & Earn UI**
- New section in account settings: "Refer & Earn"
- Shows: user's code, shareable URL, copy button, successful referral count, credit balance
- Shows milestone progress: "X/3 referrals to earn a free month"

*Acceptance criteria*
- [ ] Copy button copies `https://quicklist.ai/?ref=CODE` to clipboard
- [ ] Referral count and credit balance update without page reload after a referral completes
- [ ] Milestone progress bar is visible and accurate

## Database Schema Changes

```sql
-- 1. Add referral_code to users table
ALTER TABLE users
  ADD COLUMN referral_code VARCHAR(12) UNIQUE,
  ADD COLUMN milestone_reward_issued BOOLEAN NOT NULL DEFAULT FALSE;

CREATE UNIQUE INDEX idx_users_referral_code ON users(referral_code);

-- 2. Referrals table — tracks each successful referral relationship
CREATE TABLE referrals (
  id               SERIAL PRIMARY KEY,
  referrer_id      INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  referred_id      INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  code_used        VARCHAR(12) NOT NULL,
  status           VARCHAR(20) NOT NULL DEFAULT 'complete', -- 'pending' reserved for future async flows
  rewarded_at      TIMESTAMPTZ DEFAULT NOW(),
  created_at       TIMESTAMPTZ DEFAULT NOW(),

  CONSTRAINT referrals_referred_unique UNIQUE (referred_id), -- one referral per user
  CONSTRAINT referrals_no_self_referral CHECK (referrer_id <> referred_id)
);

CREATE INDEX idx_referrals_referrer ON referrals(referrer_id);

-- 3. Credit wallet — append-only ledger
CREATE TABLE user_credits (
  id          SERIAL PRIMARY KEY,
  user_id     INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  amount      INTEGER NOT NULL,                    -- positive = credit, negative = debit
  reason      VARCHAR(100) NOT NULL,               -- 'referral_signup', 'referral_reward', 'milestone', 'generation'
  expires_at  TIMESTAMPTZ DEFAULT (NOW() + INTERVAL '12 months'),
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_user_credits_user ON user_credits(user_id);

-- Helper view: current balance per user
CREATE VIEW user_credit_balance AS
  SELECT
    user_id,
    SUM(amount) AS balance
  FROM user_credits
  WHERE expires_at > NOW()
  GROUP BY user_id;
```

## API Endpoints

### GET /api/referral/code
Returns the authenticated user's referral code and stats. Creates a code if none exists.

**Response 200**
```json
{
  "code": "DEAN47",
  "shareUrl": "https://quicklist.ai/?ref=DEAN47",
  "successfulReferrals": 2,
  "milestoneTarget": 3,
  "milestoneRewardIssued": false
}
```

---

### POST /api/referral/complete
Called by frontend after Clerk signup, submitting the captured ref code from localStorage.

**Request body**
```json
{ "code": "DEAN47" }
```

**Response 200** — referral processed
```json
{
  "success": true,
  "creditsEarned": 5,
  "message": "You've received 5 bonus generations!"
}
```

**Response 400** — invalid code
```json
{ "error": "Invalid referral code" }
```

**Response 400** — self-referral
```json
{ "error": "You cannot use your own referral code" }
```

**Response 409** — already referred
```json
{ "error": "A referral has already been applied to this account" }
```

## Credit Redemption Logic (generate endpoint)

Replace the current quota check in `/api/generate` with this ordered logic:

```javascript
async function checkAndDeductGeneration(userId, pool) {
  return await pool.query('BEGIN');

  try {
    // 1. Check non-expired credit balance
    const creditResult = await pool.query(
      `SELECT COALESCE(SUM(amount), 0) AS balance
       FROM user_credits
       WHERE user_id = $1 AND expires_at > NOW()`,
      [userId]
    );
    const creditBalance = parseInt(creditResult.rows[0].balance);

    if (creditBalance > 0) {
      // Deduct one credit (FIFO — deduct from oldest non-expired first)
      await pool.query(
        `INSERT INTO user_credits (user_id, amount, reason, expires_at)
         VALUES ($1, -1, 'generation', NOW() + INTERVAL '12 months')`,
        [userId]
      );
      await pool.query('COMMIT');
      return { allowed: true, source: 'credits', remaining: creditBalance - 1 };
    }

    // 2. Fall through to monthly quota
    const usageResult = await pool.query(
      `SELECT generations_used, monthly_limit
       FROM users WHERE id = $1`,
      [userId]
    );
    const { generations_used, monthly_limit } = usageResult.rows[0];

    if (generations_used >= monthly_limit) {
      await pool.query('ROLLBACK');
      return { allowed: false, source: 'quota', remaining: 0 };
    }

    await pool.query(
      `UPDATE users SET generations_used = generations_used + 1 WHERE id = $1`,
      [userId]
    );
    await pool.query('COMMIT');
    return { allowed: true, source: 'quota', remaining: monthly_limit - generations_used - 1 };

  } catch (err) {
    await pool.query('ROLLBACK');
    throw err;
  }
}
```

## Referral Code Generation Logic

```javascript
async function generateReferralCode(user, pool) {
  const base = (user.name || user.email)
    .split(' ')[0]
    .replace(/[^a-zA-Z]/g, '')
    .substring(0, 6)
    .toUpperCase();

  let code, attempts = 0;
  do {
    const suffix = String(Math.floor(Math.random() * 90) + 10); // 10–99
    code = `${base}${suffix}`;
    const existing = await pool.query(
      'SELECT id FROM users WHERE referral_code = $1', [code]
    );
    if (existing.rows.length === 0) break;
    attempts++;
  } while (attempts < 10);

  await pool.query(
    'UPDATE users SET referral_code = $1 WHERE id = $2',
    [code, user.id]
  );
  return code;
}
```

Call this in the Clerk `user.created` webhook handler.

## Frontend Changes

### localStorage capture (add to index.html on page load)
```javascript
(function() {
  const params = new URLSearchParams(window.location.search);
  const ref = params.get('ref');
  if (ref && /^[A-Z0-9]{4,12}$/.test(ref)) {
    const entry = { code: ref, ts: Date.now() };
    localStorage.setItem('ql_ref', JSON.stringify(entry));
  }
})();
```

### Post-signup referral submission (call after Clerk signup completes)
```javascript
async function submitReferralCode() {
  try {
    const raw = localStorage.getItem('ql_ref');
    if (!raw) return;
    const { code, ts } = JSON.parse(raw);
    if (Date.now() - ts > 30 * 24 * 60 * 60 * 1000) {
      localStorage.removeItem('ql_ref');
      return;
    }
    await fetch('/api/referral/complete', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ code })
    });
    localStorage.removeItem('ql_ref');
  } catch (e) {
    // silent fail — don't block signup
  }
}
```
